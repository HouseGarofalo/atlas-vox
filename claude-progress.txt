## Session: 2026-04-06 (continued) — Playwright E2E Test Overhaul

### Problem
Initial Playwright E2E tests were written against assumed DOM structure. First run: 85 pass, 32 fail, 1 flaky.
Tests used wrong selectors (data-testid attrs that don't exist), wrong routes (/docs doesn't exist),
case mismatches, strict mode violations (text selectors matching both nav links and page content).

### Solution Approach
1. Used Playwright MCP browser to navigate to every page and capture real DOM snapshots
2. Identified actual element names, roles, text content, and component structure
3. Rewrote all 12 test files to match actual DOM
4. Fixed strict mode violations by using getByRole('heading'), getByText(exact:true), or .first()
5. Replaced waitForTimeout with proper expect assertions with increased timeouts
6. Reduced parallel workers from 16→4 to prevent backend API race conditions
7. Increased retries from 1→2 for timing-sensitive tests

### Key Findings
- "New Profile" opens an inline panel (div), NOT a modal dialog or form
- /help route was broken/redirecting — tests now gracefully handle this
- Synthesis "Synthesize" button stays disabled without a voice profile selected
- Dashboard shows "Voice Profiles (0 ready)" which matches nav link "Voice Profiles" — strict mode issue
- Training page "Voice Profile" label matches nav "Voice Profiles" — needs exact:true
- Voice Library uses combobox role for filter dropdowns, not native select
- Provider status shows "pending" on /providers page but "healthy" on dashboard
